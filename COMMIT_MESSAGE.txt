Refactor: Modularize monolithic main.py into clean architecture

BREAKING CHANGES: None (100% backward compatible)

## Summary

Refactored the 2010-line monolithic `src/app/main.py` into a modular
architecture with separate files for configuration, utilities, and pages.
Main entry point reduced to 94 lines (95.3% reduction) while preserving
all functionality.

## Changes

### File Structure
- Created `app/config/` for constants and configuration
- Created `app/utils/` for helper functions and utilities
- Created `app/pages/` for page rendering modules
- Created `app/components/` for future UI components (stub)
- Backed up original to `main_original_backup.py`

### New Modules (18 files, 2,502 lines total)

**Config (78 lines):**
- `config/constants.py`: File paths, OVERLAP_MAP, SECRETARIES, SLOT_START_HOURS
- `config/__init__.py`: Module exports

**Utils (412 lines):**
- `utils/helpers.py`: word_in_text, format_duration, is_event_in_window
- `utils/data_loaders.py`: get_game_data, get_special_events, get_daily_templates, get_active_tasks
- `utils/task_manager.py`: Task lifecycle (cleanup, window queries, activation count)
- `utils/secretary.py`: Secretary event persistence
- `utils/time_utils.py`: setup_timezone_and_time() - centralized time context
- `utils/__init__.py`: Module exports

**Pages (1,918 lines):**
- `pages/dashboard.py`: Strategic Dashboard with 24-hour optimization plan (686 lines)
- `pages/weekly_calendar.py`: Weekly 2× opportunities calendar (226 lines)
- `pages/arms_scheduler.py`: Arms Race weekly scheduler (182 lines)
- `pages/vs_duel.py`: VS Duel event manager (115 lines)
- `pages/special_events.py`: Special events configuration (123 lines)
- `pages/daily_tasks.py`: Task templates and activation (299 lines)
- `pages/calculator.py`: Speed-up calculator (116 lines)
- `pages/secretary_buffs.py`: Secretary buffs tracking (150 lines)
- `pages/__init__.py`: Module exports

### Refactored main.py (94 lines)
- Import all utilities and page modules
- Setup page config and data directory
- Render sidebar and time context via setup_timezone_and_time()
- Load game data and cleanup expired tasks
- Simple page routing to modular render functions

## Benefits

### Immediate
- **Maintainability:** Each page is 100-700 lines instead of buried in 2010
- **Readability:** Clear separation of concerns (config, utils, pages)
- **Testability:** Can unit test utilities without running Streamlit
- **Code Organization:** Logical grouping by functionality

### Long-term
- **Extensibility:** Easy to add new pages or utilities
- **Team Development:** Multiple devs can work on different modules
- **Reusability:** Utilities shared across pages
- **Type Safety:** Can add type hints to isolated functions

## Critical Preservation

All critical game logic preserved:
- Slot calculation: `current_slot = (now_server.hour // 4) + 1`
- Slot boundaries: [0, 4, 8, 12, 16, 20] server hours
- Timezone architecture: UTC+0 storage, UTC-2 server, user-selectable local
- Game day reset: 00:00 server time (= 22:00 Halifax)
- 24-hour optimization table with blue NOW indicator
- Secretary buff countdown and auto-expiry
- Active task tracking with daily reset at 02:00 server time
